MameServer Module¶

The MameServer module contains a class of the same name. It is the first stop in a message processing chain. It receives messages from the ‘server’ running in a MAME instance and pushes them to a queue for workers to pull from.

Examples¶

The MameServer Class¶

class cps2zmq.gather.MameServer.MameServer(front_addr, toworkers, log_to_file=False)¶

MameServer receives messages sent by an instance of MAME, and passes it to workers for processing.

context¶

zmq.Context – required by ZMQ to make the magic happen.

port¶

str – the port the serversub socket binds to.

serversub¶

zmq.Context.socket – A zmq socket set to SUB. MameClients connect and send messages here.

toworkers¶

str – the address to push work out on

backend¶

zmq.Context.socket – A zmq socket set to ROUTER. Routes work to the worker that requested it.

backstream¶

zmq.eventloop.zmqstream.ZMQStream – Used for registering callbacks with the backend socket.

msgs_recv¶

int – Total number of messages received.

workers¶

list of threads – Pool to keep track of workers.
